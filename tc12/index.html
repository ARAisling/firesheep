"Session hijacking"

  • "sidejacking"

How to stay safe?

  • Lots of things work against you.

Avoiding insecure sites when on wifi?

  • Good luck.

Avoiding insecure sites when on wifi?

  • Doesn't matter in the end.
  • Active attacker can redirect your browser.
    (SSLStrip)

Log out

  • But don't forget.
  • Logging out doesn't guarantee session is invalidated.

Use only secure wifi

  • WPA2 designed to protect clients from each other.
  • No more coffee shops for you.
  • Wifi has long history of problems. (Hole 196)
  • Do you trust the network operator?
  • What about ISP?

VPN

  • Good, but:
  • Doesn't come up right away.
  • What happens when it drops?
  • Pushes problem to the other side.
  • Slow, annoying.

Need end-to-end encryption

  • Traffic should be private between you and destination.
  • Other solutions just move problem, don't fix.

Excuses:

1. Ignorance

2. IP Addresses

  • Server Name Indication (RFC 3546)
  • Allows virtual hosting with SSL/TLS
  • Firefox 2+, Opera 8+, IE 7+, Chrome, Safari, iPhone, Android

    (XP users: Firefox only)

3. Scaling

Gmail

  • Went full SSL in January 2010.
  • "we had to deploy no additional machines and no special hardware."

    - http://www.imperialviolet.org/2010/06/25/overclocking-ssl.html
  • If you can't scale securely, you're not scaling!

Web 2.0 Privacy Concerns
